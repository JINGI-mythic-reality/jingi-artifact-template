JINGI Proof of Authorship

このリポジトリを JINGI アプリに登録するには、
GitHubアカウント所有者本人による証明（Proof） が必要です。

⸻

1. Proofの目的
	•	他人の成果を自分の創作として登録する行為を防ぐ
	•	GitHub上で実際に編集権限を持つことを確認する
	•	ゲーム内に反映される成果の信頼性を担保する

⸻

2. 証明方法（必須）

使用するファイル
	•	ファイル名：jingI-proof.txt
	•	置き場所：リポジトリのルート直下
	•	対象ブランチ：default branch（main または master）

ファイル内容
	•	JINGIアプリが発行した nonce（ランダム文字列） をそのまま1行で記載してください

例：

jingi-proof: 9f82a3d1-7c4b-4e12-b0c9-acde12345678

⚠ 内容の加工・追記・改行は行わないでください

⸻

3. 登録手順（参加者向け）
	1.	JINGIアプリで GitHubアカウントを連携する
	2.	アプリ上に表示された nonce（証明用文字列）をコピーする
	3.	本リポジトリに jingI-proof.txt ファイルを作成する
	4.	nonce を貼り付けて commit & push する
	5.	JINGIアプリに戻り、このリポジトリのURLを登録する

補足
	•	GitHub Web UI / CLI どちらの方法でも問題ありません
	•	登録後、サーバ側で自動検証が行われます

⸻

4. 共同制作・コラボレーションについて
	•	本リポジトリに push 権限 を持つメンバーは、成果登録が可能です
	•	複数人による制作の場合、ゲーム内では「共同貢献」として表示されます
	•	fork元リポジトリではなく、実際に編集権限を持つリポジトリ が登録対象です

⸻

5. 無効となるケース

以下の場合、登録は無効または保留になります。
	•	過去に使用された nonce を再利用している
	•	他人の nonce をコピーしている
	•	jingI-proof.txt が default branch に存在しない
	•	fork 先で proof を行い、fork 元を登録しようとしている
	•	明らかに成果と関係のないリポジトリ

⸻

6. 不正・疑義がある場合の対応
	•	運営は、追加の確認や説明を求める場合があります
	•	疑義が解消されるまで、成果は一時的に非公開または限定反映となることがあります
	•	明らかな悪意・不正が確認された場合、登録の取消・アカウント制限を行うことがあります

⸻

7. 本Proofの位置づけ

このProofは、
	•	創作の価値を守るための最低限の仕組みであり
	•	参加者同士の信頼を支えるための基盤です

形式的な厳格さよりも、
誠実な創作と共有が正しく評価される場 を目指しています。
